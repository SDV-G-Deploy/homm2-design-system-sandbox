System Layers

Tokens Color, type, spacing, shape, shadow and semantic roles. Tokens never decide screen choreography.
Components and recipes Shared primitives supply grammar; recipes decide hierarchy, allowed neighbors and collapse order.
Responsive contracts First-screen budgets, above-fold obligations, downgrade order, tap floors and label floors.
Page-local composition Backdrop, section order, copy rhythm, exact proportions and story-specific sequencing only.

Recipe Contract

One screen, one primary recipe Do not hybridize menu, dossier and ritual cores inside the same first-screen frame. Support patterns may appear, but one recipe owns hierarchy.
Required beats stay visible If a recipe requires a primary command or reading field, mobile collapse may not hide or miniaturize it before optional layers are reduced.
Bounded image policy still applies Art may only enter through approved slots like .menu-art-layer and must remain atmospheric, never structural or text-bearing.
Page CSS cannot rewrite system order Local composition may place and size recipe parts, but it must not promote tertiary rails above the primary command or reading task.

Responsive Contracts

Responsive behavior belongs between recipes and page-local composition. Local CSS may choose exact proportions, but it must not re-decide what stays above the fold or shrink labels below legibility.

Menu Shell Contract

Reusable menu anatomy stays in shared CSS. Page files may arrange these pieces, but should not redefine command weight, rail grammar or art-slot behavior.

Core shell .menu-screen-shell wraps .menu-frame-shell; page CSS owns only backdrop, stage proportions and grid placement.
Primary mount .menu-center-mount and .command-slab own title and command hierarchy. Rails remain support.
Support layers .menu-side-rail, .menu-save-slot, .menu-route-index and .menu-art-layer downgrade before the command slab.

Proof path: MenuScreenRecipe fixture and current operator index.

Art Slot + Fallback Rules

Image assets are optional assists. The component must remain readable, ordered and useful when an image fails, is removed or is replaced by a theme.

No-text ruleReadable labels, state names, counts and destinations stay in HTML/CSS.
Optional layerRemoving .menu-art-layer must leave a complete menu shell.
Bounded slotsDo not add art outside named component slots without a new contract row.
Frame slotsmenu-frame-corner, menu-frame-edge-x, and menu-frame-edge-y may add frame weight, never layout or labels.
Route badge slotroute-badge-sprite may supply small destination marks; route text remains HTML.
Strategy sourceUse asset-layer pilots and the operator index for proof status.

Compact Do / Do Not

Do - command hierarchyOne ceremonial command leads; quiet commands support or exit.
Do not - command hierarchyDo not place several gold commands with equal visual pressure in one slab.
Do - side rail densityKeep rails scan-friendly with compact groups and muted secondary state.
Do not - side rail densityDo not let a rail become a second command slab or a dense dossier.
Do - route indexUse route badges as small anchors beside HTML route labels.
Do not - route indexDo not make route choices compete with the main journey command.
Do - parchment fieldsUse parchment for reading, record keeping and comparison.
Do not - parchment fieldsDo not use parchment panels as decorative backing for every control.

Responsive Acceptance Checks

First screenVerify each recipe at 1440 x 900 and 390 x 844 against its budget row.
Collapse rankOptional art, rails, notes and annotations yield before primary tasks shrink.
Control floorInteractive controls stay at 44px preferred or 40px compact minimum.
Label floorScan labels stay at 12px or larger; no mobile fix may rely on micro-text.
Utility nav plaquesShort shell-nav labels stay one visual line; use horizontal scroll, summary or collapse before mid-word breaks.
Indexed feed rowsNumeric step badges stay in a stable 38-44px square track and cannot steal the row title column.
Semantic accentsUse mana, moss, wax and verdigris for state/focus meaning; do not let stateful pages collapse into only brown, parchment and gold.
Overflow smokeCheck 390 x 844 for page-level horizontal overflow; only intentional code/documentation scrollers may exceed the viewport.

Art Asset Layer

Generated imagery is allowed only as a bounded component slot. CSS still owns structure, states, copy, responsive behavior and fallback rendering.

CSS-only first Layout, rails, buttons, labels, states and adaptive compression never move into images.
Image-assisted slots Use generated art only for named slots like frame corners, edge tiles, parchment grain or route badges.
Fallback is mandatory Every slot must degrade to the existing component without losing hierarchy or readability.
Slot contract menu-frame-corner, menu-frame-edge-x, menu-frame-edge-y, route-badge-sprite. See the Art Slot + Fallback Rules for ownership and responsive checks.
Generation rule No readable text, no layout baked into art, no one-off ornaments without an owning component.
Pilot rule Frame corners and edge tiles are live. Route badges are now demonstrated as a small optional sprite slot.
Route badge sprite · optional asset layer

Page Recipes

This is the operator map. Detailed proof stays in fixtures and current pass reports.

MenuScreenRecipe

Menu-first entry screen for chapter selection, world entry and primary command routing.

  • Key classes: .menu-screen-shell, .menu-frame-shell, .menu-center-mount, .command-slab.
  • Mobile: route index and rails downgrade before command slab and save/status.
  • Proof: MenuScreenRecipe fixture and current operator index.

DossierReadingRecipe

Reading-first archive or dossier screen for records, comparison and retained lore.

  • Key classes: .parchment-object, .archive-rail, notes or metadata strips as support.
  • Mobile: notes, side matter and metadata compress before prose and reading measure.
  • Proof: DossierReadingRecipe fixture and current proof index.

ArticleHeroRecipe

Content-first first screen for longform or applied pages with one clear reading/launch beat.

  • Key classes: article hero host, primary frame, support rail only when it does not outrank the hero.
  • Mobile: hero reading task stays first; support notes move after the primary field.
  • Proof: ArticleHeroRecipe fixture and current proof index.

FeedListingRecipe

Scan-first listing for ordered routes, indexed rows and compact outcome comparison.

  • Key classes: feed host, stable row track, optional .feed-row-index markers.
  • Mobile: row badges keep a stable square track; route titles keep the readable column.
  • Proof: FeedListingRecipe fixture with indexed route rows.

RitualThresholdRecipe

Consequence-first threshold for commitment, initiation, warning or final choice.

  • Key classes: .ritual-panel, one .btn-ceremonial, one quiet escape.
  • Mobile: aura and protocol compress before actions or warning copy.
  • Status: manual recipe; add fixture proof before promoting broad claims.

RouteBoardRecipe

Stateful route progression board that shows what is current, locked, visited and next.

  • Key classes: .menu-route-index, .route-badge, compact state labels.
  • Mobile: annotations and filters downgrade before current/next state rows.
  • Status: manual recipe; use FeedListing or MenuScreen fixtures for adjacent proven behavior.

Accessibility Baseline

KeyboardAll real links/buttons use :focus-visible. Visual state specimens stay out of the tab order.
Mobile tap targetsCommands prefer 44px min-height; secondary controls may compact to 40px; labels stay 12px or larger.
ContrastQA gold-on-parchment and mana-on-dark after every theme override.
MotionReduced-motion is global; page CSS must not reintroduce mandatory animation.
