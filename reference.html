How to use this manual

1 · Choose a material role Parchment for reading, stone/night for structure, gold for the primary command, mana only for focus or consequence.
2 · Assemble from components Use buttons, rails, parchment panels, plaques, and ritual panels before writing local scene rules.
3 · Compose locally Scene/page CSS may set layout, backdrop, and dramaturgy. It must not invent new button, rail, or panel grammar.

Frame Anatomy

Outer metal rail Sections should read like mounted artifacts: a visible perimeter, a darker holding shadow, then the content field inside.
Inner parchment / stone field Content should sit inside a second contour. One border is not enough for the HOMM2 direction.
Header · body · footer Panels need anatomy. Labels, separators, and action rails should feel assembled, not poured into one soft rectangle.

Shape Language

The current direction is intentionally less round: interactive elements should feel like plaques, panels should feel framed, and hero surfaces should feel architectural. Pills/circles are exceptions.

Layout Rhythm

Less centered softness Prefer anchored copy blocks, rails, asymmetry, and structural offsets over generic centered landing composition.
Section anatomy Panels should read as assembled artifacts with header, body, and footer/action zones instead of one continuous soft block.
Archive density Use compact labels, separators, and grouped information so the interface feels authored and archival, not sparse and startup-like.

Pixel / Old-Style Motifs

Stepped edge cues Not literal retro cosplay, but tiny stepped cuts, frame joints, and beveled corners should hint at pixel-era construction.
Chunky CTA silhouette Buttons should feel pressed from metal or carved from painted wood, not inflated like modern rounded pills.
Editorial-fantasy mix Borrow archival pacing and framed presentation from portfolio/editorial references, then ground it in our fantasy materials and tokens.

Ornament Discipline

Use sigils as anchors Brand sigils, plaques, and wax seals should mark entry, ownership, or consequence — not decorate every container.
Frame before flourish Always build hierarchy with rails, insets, headers, and command depth before adding heraldic detail.
Magic is a state, not wallpaper Glow, mana blue, and ceremonial shimmer belong to focus, action, and transformation moments only.

Page Recipes

Main menu / chapter select

  1. Use one dominant .game-menu-stack or plaque command group.
  2. Move status rails and route indexes below or beside it at lower contrast.
  3. Keep gold on the single start/enter command; keep mana off until threshold states.

Archive / dossier screen

  1. Place reading content on .parchment-object.
  2. Use .archive-rail for chapters and .note-card for secondary lore.
  3. Do not turn every annotation into a card grid; preserve a route or reading spine.

Threshold / ritual screen

  1. Reserve .ritual-panel for the final commitment beat.
  2. Pair one quiet escape with one ceremonial command.
  3. Use mana glow as consequence/focus, never page wallpaper.

Accessibility Baseline

KeyboardAll links/buttons use the shared :focus-visible ring. Do not remove outline locally.
Mobile tap targetsCommand controls should stay near 40px min-height; micro labels must not drop below readable caption scale.
ContrastGold-on-parchment and mana-on-dark require visual QA after any theme/faction override.
MotionReduced-motion mode is global; page CSS should not reintroduce long mandatory animation.

Next Passes

  1. Stress-test the mobile plaque rail in real devices: current pass prevents random two-row wrap, but thumb ergonomics still need visual QA.
  2. Promote frame anatomy into reusable primitives: Panel, Frame, Crest Header, Seal, Threshold, Sigil, Banner.
  3. Test a restrained heraldic sub-mode with harder outlines and stronger step-edge cues, but keep modern typography and interaction affordance.
  4. Introduce artifact navigation variants: chapter plaque rail, council tabs, framed active state, compact world-map sidebar logic.
  5. Codify ornament & consequence as data attributes: data-threshold, data-weight="ceremonial", data-crest.
  6. Accessibility audit pass on the gold-on-parchment and mana-on-dark pairings, especially after glow restraint.
