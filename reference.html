Frame Anatomy

Outer metal rail Sections should read like mounted artifacts: a visible perimeter, a darker holding shadow, then the content field inside.
Inner parchment / stone field Content should sit inside a second contour. One border is not enough for the HOMM2 direction.
Header · body · footer Panels need anatomy. Labels, separators, and action rails should feel assembled, not poured into one soft rectangle.

Shape Language

The current direction is intentionally less round: interactive elements should feel like plaques, panels should feel framed, and hero surfaces should feel architectural. Pills/circles are exceptions.

Layout Rhythm

Less centered softness Prefer anchored copy blocks, rails, asymmetry, and structural offsets over generic centered landing composition.
Section anatomy Panels should read as assembled artifacts with header, body, and footer/action zones instead of one continuous soft block.
Archive density Use compact labels, separators, and grouped information so the interface feels authored and archival, not sparse and startup-like.

Pixel / Old-Style Motifs

Stepped edge cues Not literal retro cosplay, but tiny stepped cuts, frame joints, and beveled corners should hint at pixel-era construction.
Chunky CTA silhouette Buttons should feel pressed from metal or carved from painted wood, not inflated like modern rounded pills.
Editorial-fantasy mix Borrow archival pacing and framed presentation from portfolio/editorial references, then ground it in our fantasy materials and tokens.

Next Passes

  1. Faction themes — Knight, Sorceress, Warlock, Wizard, Barbarian, Necromancer — as overrides of the semantic layer only.
  2. HOMM1 heraldic sub-mode — harder outlines, blockier color, stronger pixel-step edges — as [data-mode="heraldic"].
  3. Real component layer with documented props and states: Panel, Frame, Seal, Threshold, Sigil, Banner, Hoard, Scroll.
  4. Introduce artifact navigation patterns: tab rail, chapter plaque, framed active state, compact world-map sidebar variants.
  5. Codify ornament & consequence as data attributes: data-threshold, data-weight="ceremonial".
  6. Accessibility audit pass on the gold-on-parchment and mana-on-dark pairings.
